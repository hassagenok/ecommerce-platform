# Repository hygiene:
Добавил в .gitignore все нужные path для того, чтобы гит не предлагал их добавлять в tracked.

* The following files should be committed to the repository:

- Source code — основной код проекта, необходимый для сборки и запуска приложения.
- Tests — автоматические проверки, которые помогают убедиться в правильности работы кода.
- Documentation (docs) — инструкции и описания, которые помогают понять проект и работать с ним.
- Makefile — файл с командами для автоматизации сборки, тестирования и управления проектом.
- .env.example — пример файла с переменными окружения без хранения секретных данных.
- CHANGELOG — список важных изменений между версиями проекта.
- Release notes — описание новых функций, исправлений и изменений в конкретном релизе.

* The following files should not be committed to the repository:

- Real secrets — пароли, API-ключи, токены и конфиденциальные данные, которые могут привести к утечке доступа.
- Local .env files — локальные файлы с настройками окружения и личными значениями, которые не должны храниться в репозитории.
- Logs — файлы с логами работы приложения.
- Cache — временные данные, созданные программами для ускорения работы, но не необходимые для запуска проекта.
- Generated artifacts without an explicit reason — автоматически созданные файлы, которые не нужно добавлять без конкретной необходимости.

# Already tracked files:

Если файл находится на стадии tracked, то файл все равно будет отображаться, даже если мы добавим его в .gitignore. Это происходит по той причине, что .gitignore работает исключительно с untracked files.

Если файл был добавлен в репозиторий до того, как он попал в .gitignore, его нужно удалить из отслеживания Git вручную: git rm --cached

# Release checklist:

Перед созданием релиза необходимо проверить:

- git status --short — проверить список изменённых и новых файлов перед коммитом.
- git diff --check — проверить изменения на наличие лишних пробелов и проблем форматирования.
- make check — запустить проверки проекта.
- Review only intentional files — убедиться, что в коммит попадают только нужные и запланированные файлы.

# local checks:
make check
git status --short
git diff --check
