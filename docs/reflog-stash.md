# find lost position:
reflog спасает в случае, когда разработчик сделал неудачных локальный rebase или reset в feature branch. Коммиты могут стать недостижимыми из текущей ветки, а это значит что git log уже не поможет найти нужный коммит, так как он показывает историю коммитов, которые в данный момент находятся в ветке, в таком случае нужно использовать reflog, который покажет какие проводились дейстия с HEAD локально.

# safety procedures:
Безопасная процедура для git reflog выглядит таким образом:
1. git status - сначала проверяем текущее состояние, укажет: актуальная ветка, есть ли незакомиченные изменения, есть ли конфликты.
2. git reflog - находим потерянное состояние HEAD.
3. Создаем recovery branch - сохраняем найденное состояние.
4. make check - проверяем восстановление (если предусмотренно командой, обычно make check запускает тесты, линтеры и другие проверки)

# stash scenario:
Stash уместен в случае, когда нам нужно выполнить hotfix, но при этом мы не хотим делать коммит текущего рабочего состояния. Важно подметить, если hotfix затягивается, то лучше создать feature branch и сделать commit.

Stash локальный и плохо описывает намерения разработчика. При долгих hotfix теряет пользу, так как быстро забывается, сложно проводить review и сложно объяснить команде. По этой причине Stash не заменяет feature branch и git commit.
