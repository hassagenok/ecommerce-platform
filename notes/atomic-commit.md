# Почему атомарный commit содержит одну законченную мысль, почему это не равно "один файл на commit" и почему смешанный commit дороже проверять:

Атомарный коммит - это изменение, которое содержит в себе одну завершённую идею и одну причину изменения.
    Легкий review.
    Быстрый поиск ошибок.
    Чистый откат.

Атомарный коммит не зависит от количества файлов, он может включать несколько файлов, если все они относятся к одной и той же задаче или причине изменения.

Смешанный коммит объединяет разные, не связанные между собой причины и изменения.
    Это усложняет ревью, потому что нужно разбирать несколько задач одновременно и легко упустить мелкие детали.
    Усложнения в откатах, потому что нельзя откатить одну часть изменений без другой
    Создается лишний шум в истории проекта.

# Хороший пример атомарного коммита:
Documents notes add atomic commit explanation and examples - хороший атомарный коммит, потому что все изменения относятся к одной теме документации atomic commit, и следовательно у нас одна причина изменения.

# Плохой пример атомарного коммита:
update notes — плохой атомарный коммит, потому что в папке notes находятся файлы с разными темами, поэтому у коммита нет одной причины изменения.
