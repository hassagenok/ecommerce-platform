# Опишим ситуация, в которой worktree будет лучше привычного git switch:
Если в папке проекта есть незакомиченные файлы или конфликты с файлами, Git может не позволить переключиться, если это приведёт к потере или перезаписи незакоммиченных изменений, решением в таком случае будет:
1. сделать коммит незавершенного состояния проекта
2. git stash
3. клонировать репозиторий

Однако есть более удобный и чистый способ решения: worktree.
git worktree создаёт дополнительную рабочую директорию, связанную с тем же репозиторием, но с отдельным состоянием файлов и индексом. Это позволяет одновременно работать с разными ветками без переключения текущей рабочей директории и без необходимости использовать stash или commit для временного сохранения изменений.

# Пример если ветка существует:
git worktree add ../hotfix hotfix/makefile-docfix
# Пример если нужно создать ветку:
git worktree add -b hotfix/makefile-docfix ../hotfix main

# Объяснение, чем linked worktree отличается от clone:
linked worktree содержит общий storage и что главное, собственное состояние рабочей папки (working tree). Каждый linked worktree имеет отдельный индекс и working tree state.

git clone создаёт абсолютно независимою и не связанную копию репозитория, в которой:
1. отдельный working tree
2. полностью отдельный index
3. связь с оригиналом существует только через remote, если он настроен. Иначе автоматически они не синхронизированы.

Ключевое отличие:
worktree - дополнительная рабочая папка одного репозитория
clone - отдельный репозиторий

# Почему clean status в одной папке не доказывает clean status в другой:
Так как каждый worktree имеет свой index, свой working directory, свой HEAD и если мы видим git status --short пустым в одной папке, это не означает что в другой он будет так же пустым.

# Предупреждение:
feature branch после локальной проверки идёт в review, но никак не идёт напрямую в protected main.

# checklist состояния каждой рабочей папки:
pwd
git worktree list
git branch --show-current
git status --short
make check

# Локальные проверки до commit:
git status --short
git branch --show-current
git diff
git diff --staged
make history
make check

# Локальные проверки после commit: 
make check
git status --short
git branch --show-current
