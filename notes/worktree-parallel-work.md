# Опишим ситуация, в которой worktree будет лучше привычного git switch:
Если у нас в текущей папке проекта есть незакомиченные файлы или какие либо конфликты с файлами, Git не позволит переключить состояние, если это может привести к потере или перезаписи незакомиченных изменений, решением в таком случае можно использовать данные шаги решения:
1. сделать коммит незавершенного состояния проекта
2. git stash
3. клонировать репозиторий заново
Однако есть более удобный и чистый способ решения: worktree.

Поэтому, если нам нужно срочно переключиться на другое состояние проекта без последствия для текущего, то мы используем git worktree. Этим мы создаем другую рабочую папку с новой веткой. Это позволяет одновременно работать с несколькими состояниями проекта, не затрагивая текущую рабочую директорию и не прерывая незавершённую работу.

# Пример:
git worktree add -b ../hotfix hotfix/makefile-docfix main

# Объяснение, чем linked worktree отличается от clone:
linked worktree содержит общий storage и что главное, собственное состояние рабочей папки (working tree). Каждый linked worktree имеет отдельный индекс и working tree state.

git clone создаёт абсолютно независимою и не связанную копию репозитория, в которой:
1. отдельный working tree
2. полностью отдельный index
3. нет синхронизации с основным репозиторием.

Ключевое отличие:
worktree - дополнительная рабочая папка одного репозитория
clone - отдельный репозиторий

# Почему clean status в одной папке не доказывает clean status в другой:
Так как worktree имеет другой index и если мы видим git status --short пустым в одной папке, это не означает что в другой он будет так же пустым.

# Предупреждение:
feature branch после локальной проверки идёт в review, но никак не идёт напрямую в protected main.

# checklist состояния каждой рабочей папки:
pwd
git worktree list
git branch --show-current
git status --short
make check

# Локальные проверки до commit:
git status --short
git branch --show-current
git diff
git diff --staged
make history
make check

# Локальные проверки после commit: 
make check
git status --short
git branch --show-current
