# Задача ветки:
Задача ветки изолировать текущее состояние проекта от изменений в базовой ветке (main), чтобы выполнять задания независимо от основной линии.

# Мой текущий контекст:
Меняем ветку с основной (main) на новую ветку (homework-v3-03-01-branch-task-isolation) через команду git switch -c
git branch --show-current показывает текущую активную ветку: homework-v3-03-01-branch-task-isolation
После смены ветки с base branch на feature делаем проверку:
git status --short - показал пустой вывод (значит никаких незакомиченых изменений в working tree не обнаружено)
make check - минимальная проверка состояния

# Почему branch не является копией проекта:
branch это скорее новое имя в проекте, пока мы не сделали новый commit - новая ветка и основная ветка смотрят на один последний коммит, после того как мы сделаем commit в новой ветке - наша новая ветка пошла вперёд, а основное состояние по-прежнему осталось на прежнем commit до интеграции.
До нового коммит main и feature branch указывают на последний commit (A - main - feature branch - HEAD)
После нового коммита feature branch движется дальше, а main остается на прежнем commit до интеграции (main указывает на A, feature branch указывает на B, B имеет родителя A, HEAD указывает на текущую ветку feature).

# Почему не main:
Мы изолируем задачу в новой ветке для того, чтобы не загрязнять историю основного проекта. Это упрощает чтение diff, делает review более чистым и понятным, облегчает отслеживание изменений и откат в случае ошибок.

# Перед коммит делаем базовые проверки проекта:
make status
make diff
git diff --staged 
make history

# Локальные проверки:
make check
git status --short
git branch --show-current

# После commit делаем финальную проверку:
make history
git show HEAD
git status --short
